# Subdomain Rename + HQ Dashboard

**Date:** 2026-03-22
**Status:** Approved
**Scope:** Rename admin.* subdomains to backoffice.*, add hq.codyjo.com cross-site dashboard

## Problem

Dashboards currently live at `admin.yourdomain.com`. The "admin" prefix is generic and could conflict with future admin tools. The project is called "Back Office" — the subdomain should match. Additionally, there is no cross-site portfolio view that shows comparative health, risk, and coverage across all managed sites.

## Two Changes

### 1. Subdomain Rename

All `admin.*` references become `backoffice.*`:

| Current | New |
|---------|-----|
| `admin.codyjo.com` | `backoffice.codyjo.com` |
| `admin.thenewbeautifulme.com` | `backoffice.thenewbeautifulme.com` |

**Files to change:**
- `config/backoffice.yaml` — `subdomain:` values in `dashboard_targets` AND `api.allowed_origins`
- `config/backoffice.example.yaml` — same
- `dashboard/site-branding.js` — hostname regex from `/^admin\.(.+)$/` to `/^backoffice\.(.+)$/`
- `docs/LIVE-URLS.md` — documented URLs (note: `/back-office/admin/` on line 10 is a public site path, NOT a subdomain — leave it unchanged)
- `docs/CICD-REFERENCE.md` — any admin.* subdomain references
- `terraform/cd.tf` — update IAM policy ARN references that contain admin.* subdomain strings (note: bucket name ARNs like `admin-thenewbeautifulme-site` stay as-is since bucket names don't change)

**Files that do NOT need changes** (admin.* origins come from config YAML, not hardcoded):
- `backoffice/server.py` — defaults to localhost only
- `backoffice/api_server.py` — defaults to localhost only

**What does NOT change:**
- S3 bucket names (internal identifiers, renaming is destructive)
- CloudFront distribution IDs
- DNS records and CloudFront CNAMEs (user's responsibility — must add `backoffice.*` as alternate domain names on each CloudFront distribution before the rename takes effect)

### 2. HQ Dashboard (`hq.codyjo.com`)

A new cross-site portfolio dashboard with three sections:

#### Health Scores

One card per site showing a composite 0-100 score. Weighted average of department scores:
- QA: 25%
- ADA: 20%
- SEO: 15%
- Compliance+Privacy: 25% (compliance score at 15%, privacy score at 10% — these overlap since privacy findings are derived from compliance findings, but their scores measure different things: compliance measures GDPR/ISO overall, privacy measures personal data handling specifically)
- Product: 10%
- Monetization: 5%

Departments with no audit are excluded from the average (not counted as zero). Sites with all null scores get `health_score: null`. Color-coded: green (80+), yellow (50-79), red (<50). Default sort: worst-first (most actionable at top).

**Score extraction per department:**
- QA: derived from severity counts via `qa_score_from_summary()` (same as workflow.py)
- SEO: `summary.seo_score` field
- ADA: `summary.compliance_score` field
- Compliance: `summary.overall_score` field
- Privacy: computed via `privacy_score()` from `aggregate.py` (deduction-based formula, NOT `extract_score`)
- Product: `summary.product_readiness_score` or `metadata.product_readiness_score`
- Monetization: `summary.monetization_readiness_score` or `metadata.monetization_readiness_score`

#### Risk View

Table of findings per site sorted worst-first. Columns: site name, critical count, high count, medium count, low count, "Open in backoffice →" link. Row highlighting for sites with critical findings. Clicking a row filters the dashboard to that site.

#### Coverage Matrix

Grid with sites as rows, departments as columns. Each cell shows last audit date as relative time ("2d ago", "3w ago") with color-coded background:
- Green: <7 days
- Yellow: 7-30 days
- Red: >30 days
- Gray "—": never audited

#### Site Switcher Dropdown

At the top of the page. Default: "All Sites" (aggregated view). Selecting a site:
- Filters all three sections to that site's data in-place
- Sets `?site=name` query param
- Each site card/row also has an "Open in backoffice →" link to `backoffice.yourdomain.com`

"All Sites" resets to the aggregated view.

**Health score formula:** Weighted average of available department scores per the weights above. `totals.health_score` is the simple average of all sites' individual health scores (sites with `null` health are excluded).

**Coverage data:** Extracted from each department's findings file `scanned_at` / `timestamp` / `metadata.auditDate` field (same extraction logic already in `backoffice/workflow.py:extract_scanned_at`).

**Backoffice URL:** Constructed from `org-data.json` product metadata or derived from target name: `https://backoffice.{domain}`.

## Aggregate Module Changes

Add `aggregate_hq()` function to `backoffice/aggregate.py` that:
1. Iterates all repos in results/ directory
2. For each repo, reads all department findings files
3. Extracts scores per department (reuses existing `extract_score` logic from workflow.py or equivalent)
4. Extracts `scanned_at` timestamps for coverage
5. Counts severities for risk view
6. Computes weighted health score
7. Writes `dashboard/hq-data.json`

Called from `aggregate()` main orchestrator alongside existing department aggregation.

## Error Handling for `aggregate_hq()`

Follows the same patterns as existing aggregation (DX overhaul spec Section 5):
- **Empty results/ directory:** Write `hq-data.json` with empty `sites` and zero `totals`.
- **Findings file missing for a department:** Skip silently — cell shows `null` in coverage.
- **Malformed findings JSON:** Log warning, skip that file, continue.
- **All scores null for a site:** Set `health_score: null` (not 0). Exclude from `totals.health_score` average.
- **Score extraction fails for a department:** Log warning, that department excluded from health score weight.
