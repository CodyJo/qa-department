Back Office Documentation

CI/CD best practices reference architecture

Use this page as the Back Office delivery standard across the estate: every push should get validated, every release should be reviewable, and production should stay protected.

HQ Dashboard Admin Dashboard

Required workflows

  • CI: every push and pull request, including lint, typecheck, tests, and build.
  • Preview: PR-safe preview validation before merge.
  • Nightly: unattended release-readiness and audit refresh jobs.
  • CD: production deployment only from main, behind protected environments.

Release guardrails

  • Use concurrency to cancel stale CI and preview runs.
  • Add timeout limits to deploy and nightly jobs.
  • Run smoke tests after production deployment.
  • Prefer overnight PR creation over unattended production release.

Reference flow

Back Office audit
  -> prioritized work
  -> branch or PR
  -> CI gate
  -> preview validation
  -> review
  -> production deploy
  -> smoke test
  -> dashboard refresh

The operating principle is simple: automate analysis and verification aggressively, automate production promotion carefully.

CLI

python3 scripts/backoffice-cli.py list-targets
python3 scripts/backoffice-cli.py audit --target bible-app --departments qa,product
python3 scripts/backoffice-cli.py refresh
python3 scripts/backoffice-cli.py deploy
python3 scripts/backoffice-cli.py scaffold-workflows --target bible-app

This CLI is the portable entry point so any operator or coding agent can run Back Office the same way.
