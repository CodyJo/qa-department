# Back Office

A portable static dashboard suite for auditing and operating websites, powered by [Claude Code](https://claude.com/claude-code). Deploy the HTML dashboards into an admin subdomain or S3 bucket to review QA, SEO, ADA, compliance, monetization, product, and self-audit data generated by the agents.

## What It Does

**Public view** (clients & photographers see):
- Report a Bug form

**Admin view** (behind admin key):
- QA Dashboard — automated scan findings, severity stats, fix progress
- Tests — test run history, testing plan, UAT step management
- Tasks — TODO tracking with categories, priorities, filters
- System — infrastructure overview, health status

## How It Works

```
┌─────────────┐     findings.json     ┌─────────────┐
│  QA Agent   │ ───────────────────>  │  Fix Agent  │
│  (scanner)  │                       │  (worktrees) │
└──────┬──────┘                       └──────┬──────┘
       │                                     │
       │  Scans code for:                    │  For each finding:
       │  - Security vulns                   │  - Creates git worktree
       │  - Input validation                 │  - Applies fix
       │  - Performance issues               │  - Runs tests + linter
       │  - Code quality                     │  - Merges if green
       │                                     │
       ▼                                     ▼
  results/{repo}/                    fixes.json + commits
  findings.json                      on main branch
       │
       ▼
  ┌──────────┐
  │ Dashboard │  <── backoffice.html (static, hosted on S3/CloudFront)
  └──────────┘
```

## Quick Start

```bash
# 1. Clone and setup
git clone https://github.com/YOUR_USER/back-office.git
cd back-office
make setup

# 2. Edit config
vim config/targets.yaml    # Add your repos
vim config/qa-config.yaml  # Add your AWS settings

# 3. Run a QA scan
make qa TARGET=/path/to/your/repo

# 4. Auto-fix the findings
make fix TARGET=/path/to/your/repo

# 5. Deploy the dashboard
make dashboard
```

## Watch Mode (Overnight QA)

Set it and forget it. The watch agent monitors for new findings and auto-fixes them:

```bash
# Start watch mode (checks every 15 minutes, auto-fixes, syncs dashboard)
make watch TARGET=/path/to/your/repo

# Or customize the interval
./agents/watch.sh /path/to/repo --interval 10 --auto-fix --sync
```

## Dashboard

The repo now ships a dashboard suite in `dashboard/`, not just one page. The landing page links out to department-specific views such as `qa.html`, `seo.html`, `ada.html`, `compliance.html`, `monetization.html`, `product.html`, `jobs.html`, `self-audit.html`, and `admin.html`.

The dashboards are static HTML + vanilla JS with no build step. Data is generated by `scripts/aggregate-results.py` and written as department JSON files such as `qa-data.json`, `seo-data.json`, `ada-data.json`, `compliance-data.json`, `monetization-data.json`, `product-data.json`, and `self-audit-data.json`.

Configure deployment targets in `config/qa-config.yaml`:
```yaml
dashboard_targets:
  - bucket: "my-website-bucket"
    path: "backoffice.html"
    cloudfront_id: "EXXXXXXXXX"
```

## Configuration

### `config/targets.yaml`

Define the repositories you want to scan:

```yaml
targets:
  - name: my-app
    path: /home/user/projects/my-app
    language: python
    lint_command: "ruff check ."
    test_command: "python3 -m pytest tests/ -q"
    deploy_command: "bash scripts/deploy.sh"
    context: "Description for the QA agent to understand the project"
```

### `config/qa-config.yaml`

Configure AWS settings, scan behavior, and fix agent preferences. See `config/qa-config.example.yaml` for all options.

## Architecture

### QA Agent (`agents/qa-scan.sh`)

Launches a Claude Code session that:
1. Reads and understands the target codebase
2. Runs the project's linter and test suite
3. Performs a security audit (OWASP Top 10)
4. Checks input validation, performance, and code quality
5. Writes structured findings to `results/{repo}/findings.json`

### Fix Agent (`agents/fix-bugs.sh`)

Launches a Claude Code session that:
1. Reads the findings file
2. Filters to agent-fixable issues, sorted by severity
3. Groups findings by file to minimize merge conflicts
4. Fixes each group in an isolated git worktree
5. Runs tests and linter before merging
6. Records fix status in `results/{repo}/fixes.json`

### Dashboards (`dashboard/*.html`)

Static HTML + vanilla JS. The landing page aggregates department status, while the department pages render their own generated JSON payloads and offer local run hints for rescans.
